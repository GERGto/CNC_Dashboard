# Dokumentation

Dieser Ordner enthaelt projektrelevante Dokumentation, die ueber API- oder Implementierungsdetails hinausgeht.

## Einstieg

- [Knowledge Base](./knowledge-base.md): Produktkontext, Hardware-Zielbild, Kernfunktionen und geplante Erweiterungen.
- [Hardware](./hardware/README.md): Komponenten, Busse, Scan-Ergebnisse und Inbetriebnahme-Notizen.

## Pflege

Wenn neue Funktionen, Hardwarekomponenten oder Betriebsablaeufe hinzukommen, sollten sie in der Knowledge Base festgehalten werden.
Neue Hardwaredetails, Verdrahtung, Adressen und Verifikationsschritte gehoeren zusaetzlich nach `docs/hardware/`.
